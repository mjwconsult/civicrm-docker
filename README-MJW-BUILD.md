# Custom docker-standalong build

Currently using https://res.mjw.pt/dl/mjwcivicrm-6.3.1-standalone-6.3-std.tar.gz
which is generated by our standard build process.
(NOTE: Replace with actual version when building..)

## "Generic" build:

docker build build/civicrm --build-arg CIVICRM_VERSION=6.3.1 --build-arg PHP_VERSION=8.3 -t mjw-civicrm-6.3.1

## Our build:


So actually using "mjw" branch here: https://github.com/mjwconsult/civicrm-docker/tree/mjw

We're using standard common-base and civicrm-base but custom wordpress / standalone.
Don't need these commands:

```
docker build build/common-base -t mjw-common-base
docker build build/civicrm-base -t mjw-civicrm-base
```

For Standalone:

```
docker build --build-arg CIVICRM_DOWNLOAD_URL="https://res.mjw.pt/dl/mjwcivicrm-6.7.2-standalone-6.7-std.tar.gz" --build-arg PHP_VERSION=8.3 build/civicrm -t mjw-civicrm-6.7.2-mjw
```

For WordPress:

```
docker build --build-arg CIVICRM_DOWNLOAD_URL="https://res.mjw.pt/dl/mjwcivicrm-6.7.2-wordpress-6.7-std.zip" --build-arg PHP_VERSION=8.3 build/civicrm-wordpress -t mjw-civicrm-wordpress-6.7.2-mjw
```
